# Semantic Mapping

**Keywords:** Meaning, affordance, learning-curve, consistency.

**Description:**
Assigning specific meanings to colors within your specific product context, and sticking to them religiously.

## Usage

### 1. Reserved Colors
If "Purple" means "Premium Feature", never use Purple for a generic "Submit" button.
*   **User Learning:** Users quickly learn "Purple thing = I have to pay".

### 2. Object Mapping
In complex apps (like calendars or project management):
*   **Bug:** Red
*   **Feature:** Green
*   **Task:** Blue
*   **Epic:** Purple

### 3. Consistency
The meaning must hold true across:
*   Icons
*   Text
*   Backgrounds
*   Charts

**Rule:** Don't repurpose a semantic color for decoration. It confuses the user's mental model.
