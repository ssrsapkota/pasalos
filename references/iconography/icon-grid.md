# Icon Grid

> **Base grid (24x24, 20x20, 16x16)**

Using a grid ensures consistency in size, proportion, and alignment across an icon set.

## Common Grids
-   **24x24px:** Most common standard (Material Design, many system UI sets). Good balance of detail and legibility.
-   **20x20px:** Often used in denser interfaces.
-   **16x16px:** Small system usage (trees, compact lists).
-   **32x32px:** Large actions or feature icons.

## The Pixel Grid
-   Snap points to the pixel grid to avoid anti-aliasing fuzziness.
-   Center content within the frame.
-   Leave a "safe area" (padding) inside the frame (e.g., 2px padding in a 24px grid) to prevent optical overspill.
