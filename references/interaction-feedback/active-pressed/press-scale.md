# Press Scale

**Description:**
A slight reduction in size when an element is actively pressed (mouse down or touch start). This creates a "squishy" or tactile feel, mimicking a physical button being pushed in.

**Key Principles:**
- **Subtlety**: The shrink effect should be subtle (e.g., 0.98 or 0.95 scale). Too much shrinking looks cartoonish.
- **Speed**: The transition to the pressed state should be very fast (almost instant), while the release can be slightly slower (bounce back).
- **Feedback**: Confirms to the user that their input has been registered.

**Common Patterns:**
- **Button Press**: Standard implementation for primary and secondary actions.
- **Card Press**: Clickable cards shrink slightly.
- **Icon Press**: Navigation icons or toggle buttons.
