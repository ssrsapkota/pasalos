# Why Disabled

**Description:**
Communicating the reason *why* an element is disabled. A disabled button is a dead end; explaining how to unlock it creates a path forward.

**Key Principles:**
- **Helpfulness**: Don't just say "No". Say "No, because..."
- **Proximity**: Place the explanation near the disabled element.

**Common Patterns:**
- **Tooltips**: Hovering the disabled element shows a tooltip ("Fill in all fields to continue").
- **Helper Text**: Small text below the button ("Select a plan to subscribe").
- **Toast**: If clicking a disabled element, trigger a toast with the reason.
