# Error State

**Description:**
Visual feedback indicating that an action failed or input is invalid.

**Key Principles:**
- **Negative Color**: Red is the universal standard for error.
- **Explanation**: A visual cue alone (red border) is insufficient; you must explain *what* is wrong.
- **Accessibility**: Don't rely on color alone. Use icons (exclamation mark) and text.

**Common Patterns:**
- **Field Validation**: Red border + help text below input.
- **Toast Error**: A popup notification describing the system error.
- **Shake Animation**: A subtle horizontal shake to mimic a head shake "no".
