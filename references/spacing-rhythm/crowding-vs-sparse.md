# Crowding vs Sparse

**Concept:** Density appropriateness.

Spacing is not "one size fits all". The density of information depends on the context and the user's goal.

## Crowded (High Density)
- **Use Cases:** Data tables, financial dashboards, code editors, file managers.
- **Goal:** Maximum information visibility, seeking efficiency, comparison.
- **Spacing:** Tight (4px, 8px), small fonts (12px, 13px).

## Sparse (Low Density)
- **Use Cases:** Marketing landing pages, blogs, reading experiences, onboarding flows.
- **Goal:** Focus, readability, narrative, emotion.
- **Spacing:** Loose (32px, 64px+), large fonts (18px+).

## Spectrum
Design systems often support "modes" (Compact, Default, Relaxed) to let the user control this density, especially in productivity apps.
